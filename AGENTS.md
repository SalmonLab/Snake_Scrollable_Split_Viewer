# AGENTS.md

## 作りたいこと

Chromium系ブラウザ（現在は拡張機能ベース）で、読みやすさ重視の「横方向分割スクロール」体験を実装する。

一般的なブラウジングは縦1ストリームですが、本拡張では次を実現する。

- 画面を分割して、1ページを2つ（将来3つ）に分けて表示する。
- スクロールは画面全体で一体管理され、各分割領域が個別に独立して動かない。
- 2分割時の流れ:
  - 左エリア（1）を下まで読んだあと、右エリア（2）の上部が左エリアの下端に接続する。
  - スクロール量は左右で同期し、常に1画面分ずれる。
  - 1画面分（左画面分）進むと、右画面に新規内容が入り、左画面からは既読分が流れる。
- まずは「拡張機能として実装」。
- 追加要件が増え、拡張では限界が出たら、
  1) 独自ブラウザへ移行検討  2) 移行時の優先機能設計へ進む。

視覚要件（初期）

- 余白/レイアウト:
  - 左右の端: 余白最小（概ね 30px 程度）
  - 分割ライン: 3?5px 程度、控えめだが存在が分かる濃度


## 進め方

このルールを毎回の作業で固定で適用する。

1. タスクを番号付きで明確化し、実行中の進捗をユーザーへ提示する。
2. 各タスクは最小単位で実装し、機能単位で検証する。
3. 「実装 → 実際のブラウザで検証」は毎回必須。
4. 検証で失敗した場合は、別の実装方法で再試行する。
5. 同一方針の失敗が5回続いたら停止し、次の進め方をユーザーと相談する。


## タスク運用ルール

- 各タスクは `#1`, `#2`, ... の番号を付与して進める。
- 進行中は
  - `現在の進捗: #X / 全体` 形式で共有。
- 確認が必要な判断（例: 3分割時の優先仕様、スクロール計測位置、除外条件）は、作業前に相談する。
- 検証対象サイトは実際のブラウザ上で確認し、結果を数値/観測を含めて記録する。
- 最終成果はまず `README.md` の機能要約を更新し、実装原則・制約は本 `AGENTS.md` に積み上げる。
